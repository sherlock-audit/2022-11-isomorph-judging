# Use safeTransfer/safeTransferFrom consistently instead of transfer/transferFrom

## Summary

Use safeTransfer/safeTransferFrom consistently instead of transfer/transferFrom

## Vulnerability Detail

It is good to add a require() statement that checks the return value of token transfers or to use something like OpenZeppelinâ€™s safeTransfer/safeTransferFrom unless one is sure the given token reverts in case of a failure. 

## Impact

Failure to do so will cause silent failures of transfers and affect token accounting in contract.
